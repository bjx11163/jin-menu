设计第七部分：技术实现方案

落地细节，心里有数

🧩 前端
微信小程序原生 wxml + wxss + js

页面清单：
• 启动页（选角色）
• 菜单首页（菜品列表+分类）
• 菜品详情（大图+信息+点菜）
• 购物车
• 个人中心（她的）
• 管理首页（接单）
• 菜品编辑（增删改+图片）
• 图片编辑器
• 统计页面
• 投票/心愿/纪念日设置
☁️ 云端
微信云开发（无需服务器）

云数据库：6个集合
dishes / categories / orders / reviews / favorites / wishlist

云存储：菜品图片 / 成品照片

云函数（2-3个）：
• 提交订单 → 发订阅消息
• 纪念日检查 → 触发特餐提示
• 定时统计（可选）
🎨 样式
背景：你的复古街景图（全屏毛玻璃效果）
主色调：暖棕 #8B6B4D / 复古金 #D4AF37 / 深咖 #3E2723
卡片：毛玻璃效果 backdrop-filter: blur
字体：系统字体 + 中文优雅排版
圆角：12px-16px 大圆角
📦 部署
开发工具：微信开发者工具
小程序ID：需要注册一个微信小程序（个人主体免费）
云开发：在开发者工具里开通，一键关联
发布：微信审核（1-2天），个人主体可发
扫码：生成小程序码，她微信扫了进
💡 估算工作量：
页面约10个，核心功能6-7个。按小程序开发经验，一个人做大约 1-2周（每天2-3小时的话2-3周）。先从核心功能做起：菜单浏览 → 点菜下单 → 接单标记 → 通知，其他功能后续迭代。
