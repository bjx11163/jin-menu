设计第二部分：数据库结构

云开发数据库，几个核心集合

// 菜品集合 — 你维护
dishes {
_id, name, photo, category, description,
ingredients[], difficulty, rating(你的评分),
tags[]（辣度/口味）, createdAt, updatedAt
}
// 分类集合
categories {
_id, name (荤菜/素菜/汤/凉菜/主食), sortOrder
}
// 订单集合 — 她点菜时生成
orders {
_id, userId(她的openid), items[{dishId, name, quantity, note}],
scheduledTime(预约时间), status(pending/done),
createdAt, completedAt
}
// 评价集合 — 她吃完打分
reviews {
_id, orderId, dishId, rating(1-5), comment,
photo(你做的成品照), createdAt
}
// 收藏/心愿/投票等数据用子集合或独立集合
favorites { _id, userId, dishId }
wishlist { _id, userId, dishName, status(提议/已学/安排) }
votes { _id, createdBy, options[], expiresAt, result }

云数据库直接用 JSON 文档存，不需要建表 SQL，简单直接。
